⚠ Pain Points
  • Alur approver mobile kurang jelas — Approver tidak berhasil menyelesaikan review
  • Pengguna Form Creator ragu langkah selanjutnya setelah membuat template
  • Beberapa responden memerlukan bantuan saat memilih assignee dan mengatur due date
  • Inkonsistensi label dan tampilan antara web dan mobile
  • Status task setelah submit belum sepenuhnya meyakinkan bagi sebagian pengguna
  • Pengguna baru merasa perlu waktu belajar sebelum mandiri (SUS Q4 avg tinggi)
  • Tipe input attachment paling sering membutuhkan klarifikasi
✓ Strengths
  • Tipe input (text, date, dropdown, checkbox, rating) dinilai mudah dipahami lintas role
  • Validasi field wajib membantu pengguna menghindari kesalahan submit
  • Alur web-ke-mobile secara umum dapat diikuti tanpa banyak bantuan
  • Evidence, timestamp, dan nama assignee dinilai sangat membantu proses review
  • Mengurangi ketergantungan pada WhatsApp, Google Forms, dan spreadsheet
  • Tampilan bersih dan konsisten diapresiasi mayoritas responden
  • Status konfirmasi setelah submit dinilai jelas oleh pengguna mobile
↑ Top Priorities
  • P1 Perbaiki alur approver mobile — review & approval flow perlu redesign
  • P1 Tambahkan onboarding / guidance setelah pembuatan template
  • P2 Perjelas status task setelah submit di sisi assignee
  • P2 Tingkatkan konsistensi label web ↔ mobile
  • P2 Tambah panduan in-app untuk pengguna baru (tooltip / walkthrough)
  • P3 Perbaiki UX attachment — preview dan feedback upload lebih jelas
  • P3 Optimalkan alur multi-assignee selection pada web admin
